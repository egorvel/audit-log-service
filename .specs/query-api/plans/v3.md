# Implementation Plan: Query API on `audit_events` — Iteration 1 (v3)

## Context

PR #7 merged the spec set under `.specs/query-api/` (`requirements.md`,
`design.md`, `tasks.md`). This iteration implements the entire feature in one
branch and one PR. The deliverable is:

- `GET /api/v1/audit-events` with keyset cursor pagination, filter validation,
  and the parse-vs-semantic 400/422 error split (per all AC families in
  `requirements.md`).
- The `audit_events` table moved from V1 (numeric id, flat actor/resource) to
  V2 (ULID `id`, structured `actor_id`/`actor_type`/`resource_id`/
  `resource_type`) via Flyway table-replacement migration.
- The `POST /api/v1/audit-events` body shape updated to match (resource path
  also moves from `/api/v1/events` to `/api/v1/audit-events`).

`tasks.md` is the source of truth for what each commit must contain and prove;
this plan is the execution wrapper around it.

## Approach

One feature branch off `master`: **`feature/query-api-impl`**.

Eight commits. Commit 0 is a spec-only refinement (this file + `_delta.md`);
Commits 1–7 implement T1–T7 in dependency order:

```
C0 ── T1 ── T2 ── T4 ── T5 ── T6 ── T7
              │         │
              └── T3 ───┘
```

Each commit must keep `mvn verify` green — that is how we honor "one safe
commit per task." After Commit 7, open one PR against `master`.

### Commit 1 — T1 · `UlidFactory` + dependency

New:
- `src/main/java/com/sam/auditlog/service/UlidFactory.java` —
  `@Component` wrapping `ulid-creator`. Two methods:
  - `String next()` — monotonic-per-JVM, current time. Used by T2's
    converter for new events at runtime.
  - `String fromTimestamp(Instant)` — random-payload ULID with the given
    instant's epoch millis seeded as the time component. Used by T2's
    backfill so historical rows' ULIDs sort by their original `timestamp`.
    Backed by `UlidCreator.getUlid(long timeMillis)` from ulid-creator
    5.2.3.
- `src/test/java/com/sam/auditlog/service/UlidFactoryTest.java` —
  pure JUnit (matches existing `AuditEventConverterTest` style: no Spring).

Modified:
- `pom.xml` — add `com.github.f4b6a3:ulid-creator:5.2.3` (pinned per
  `tasks.md` T1).

No call sites use the factory yet; it is wired in T2.

### Commit 2 — T2 · V2 migration + model change + write-path move (atomic)

New:
- `src/main/java/com/sam/auditlog/db/migration/V2__query_api_model.java`
  — design §2.3 table-replacement migration as a Flyway Java migration
  (`extends BaseJavaMigration`). Single transaction:
  1. DDL: `CREATE TABLE audit_events_new (...)` per design §2.1.
  2. Backfill: stream rows from `audit_events`, INSERT into
     `audit_events_new` with `id = UlidFactory.fromTimestamp(timestamp)`,
     `actor_id = actor`, `actor_type = 'unknown'`,
     `resource_id = resource`, `resource_type = 'unknown'`, others copied
     verbatim.
  3. DDL: `DROP TABLE audit_events`,
     `ALTER TABLE audit_events_new RENAME TO audit_events`.
  4. DDL: re-create the three triggers
     (`audit_events_no_update`/`no_delete`/`no_truncate`),
     `GRANT SELECT, INSERT ON audit_events TO audit_app`, and create the
     three indexes from design §4.

  Flyway constructs Java migrations itself (no Spring DI). The migration
  instantiates a local `new UlidFactory()` rather than the Spring bean —
  both wrap the same `ulid-creator` factory, so behavior is identical.
- `src/main/resources/application.properties` — append
  `spring.flyway.locations=classpath:db/migration,classpath:com/sam/auditlog/db/migration`
  so V1 (SQL) and V2 (Java) are both discovered.
- `src/main/java/com/sam/auditlog/dto/ActorRef.java` — record
  `(String id, String type)` with `@NotBlank` on both.
- `src/main/java/com/sam/auditlog/dto/ResourceRef.java` — same shape.

Modified:
- `model/AuditEvent.java` — `Long id` → `String id`; flat `actor`/`resource`
  strings → `actorId`/`actorType`/`resourceId`/`resourceType`. Tighten
  constructor `Objects.requireNonNull` to all four. Keep
  `@Generated(INSERT)` on `timestamp`, immutability, and the existing
  defensive copy of `context`.
- `dto/CreateAuditEventRequest.java` — `actor`/`resource` become
  `ActorRef`/`ResourceRef` with `@Valid` cascade.
- `dto/AuditEventResponse.java` — design §1.3 shape; `id` becomes `String`.
- `repository/AuditEventRepository.java` — generic param to
  `<AuditEvent, String>`; remove `findAllByOrderByIdDesc` (replaced in T4).
- `service/AuditEventService.java` — remove `recent(int)` (orphaned once
  the GET endpoint goes away in this commit).
- `converter/AuditEventConverter.java` — inject `UlidFactory`; `toEntity`

- `controller/AuditEventController.java` — `@RequestMapping` path

## Changelog vs v2

- **`UlidFactory` exposes `fromTimestamp(Instant)`** in addition to
  `next()`. Needed by T2's backfill so historical rows get ULIDs whose
  time component matches their original `timestamp` (preserving sort
  order). Runtime new-event path still uses monotonic `next()`.
- **V2 migration is a Flyway Java migration** at
  `com.sam.auditlog.db.migration.V2__query_api_model` (extends
  `BaseJavaMigration`), not a `.sql` file. The INSERT-from-old can call
  `UlidFactory.fromTimestamp(...)` directly — we avoid keeping a second
  ULID generator implementation in PL/pgSQL. `application.properties`
  gains a Flyway location entry so the Java migration is discovered.
- **C0 added.** v3 introduces an explicit "Commit 0" that lands the spec
  refinement before any Java changes, so spec history doesn't drift.

# Implementation Plan: Query API on `audit_events` — Iteration 1

## Context

PR #7 merged the spec set under `.specs/query-api/` (`requirements.md`,
`design.md`, `tasks.md`). This iteration implements the entire feature in one
branch and one PR. The deliverable is:

- `GET /api/v1/audit-events` with keyset cursor pagination, filter validation,
  and the parse-vs-semantic 400/422 error split (per all AC families in
  `requirements.md`).
- The `audit_events` table moved from V1 (numeric id, flat actor/resource) to
  V2 (ULID `id`, structured `actor_id`/`actor_type`/`resource_id`/
  `resource_type`) via Flyway table-replacement migration.
- The `POST /api/v1/audit-events` body shape updated to match (resource path
  also moves from `/api/v1/events` to `/api/v1/audit-events`).

`tasks.md` is the source of truth for what each commit must contain and prove;
this plan is the execution wrapper around it.

## Approach

One feature branch off `master`: **`feature/query-api-impl`**.

Seven commits, one per task in `tasks.md`, in dependency order:

```
T1 ── T2 ── T4 ── T5 ── T6 ── T7
        │         │
        └── T3 ───┘
```

Each commit must keep `mvn verify` green — that is how we honor "one safe
commit per task." After Commit 7, open one PR against `master`.

## Critical files per commit

References use AC… (requirements) and §… (design). For full DoD see
`tasks.md`.

### Commit 1 — T1 · `UlidFactory` + dependency

New:
- `src/main/java/com/sam/auditlog/service/UlidFactory.java` —
  `@Component` wrapping a single monotonic factory instance. Exposes
  `String next()` returning the 26-char Crockford-Base32 form.
- `src/test/java/com/sam/auditlog/service/UlidFactoryTest.java` —
  pure JUnit (matches existing `AuditEventConverterTest` style: no Spring,
  no Mockito).

Modified:
- `pom.xml` — add `com.github.f4b6a3:ulid-creator` (5.x).

No call sites use the factory yet; it is wired in T2.

### Commit 2 — T2 · V2 migration + model change + write-path move (atomic)

This commit is large because schema and JPA mapping must move together.

New:
- `src/main/resources/db/migration/V2__query_api_model.sql` — design §2.3
  table-replacement migration: create `audit_events_new`, INSERT-from-old
  with derived ULID + `'unknown'` types, DROP old, RENAME, re-create
  `audit_events_no_update`/`no_delete`/`no_truncate` triggers, re-grant
  `SELECT, INSERT` to `audit_app`, create the three indexes from §4.
- `src/main/java/com/sam/auditlog/dto/ActorRef.java` — record
  `(String id, String type)` with `@NotBlank` on both.
- `src/main/java/com/sam/auditlog/dto/ResourceRef.java` — same shape.

Modified:
- `model/AuditEvent.java` — `Long id` → `String id`; replace flat `actor`/
  `resource` with `actorId`/`actorType`/`resourceId`/`resourceType`. Tighten
  constructor `Objects.requireNonNull` to all four. Keep
  `@Generated(INSERT)` on `timestamp`, immutability, and the existing
  defensive copy of `context`.
- `dto/CreateAuditEventRequest.java` — `actor`/`resource` become
  `ActorRef`/`ResourceRef` with `@Valid` cascade.
- `dto/AuditEventResponse.java` — design §1.3 shape; `id` becomes `String`.
- `repository/AuditEventRepository.java` — generic param to
  `<AuditEvent, String>`; remove `findAllByOrderByIdDesc` (replaced in T4).
- `service/AuditEventService.java` — remove `recent(int)` (orphaned once
  the GET endpoint goes away in this commit).
- `converter/AuditEventConverter.java` — inject `UlidFactory`; `toEntity`
  calls `factory.next()`; `toResponse` builds nested actor/resource.
- `controller/AuditEventController.java` — `@RequestMapping` path
  `/api/v1/events` → `/api/v1/audit-events`; remove `list(...)` (replaced
  in T6); keep `create(...)`.

Reuses (existing utilities, do **not** reinvent):
- `model/Outcome.java` + `model/OutcomeConverter.java` — unchanged.
- `controller/GlobalExceptionHandler.java` — envelope shape stays the same;
  no new handlers in this commit.
- `AuditEventConverterTest` — extend rather than rewrite; assertions about
  ULID assignment + new response shape are added there.

Test work in this commit:
- Update `AuditEventConverterTest` for the new shape.
- Update `AuditEventIntegrationTest` for the new path and body.
- Add `V2MigrationIntegrationTest` (new) — boots V1 → applies V2 → asserts:
  row count preserved; every row has 26-char ULID `id`; ordering by `id ASC`
  matches old `timestamp ASC`; `actor_type` = `resource_type` = `'unknown'`
  on historical rows; `UPDATE`/`DELETE`/`TRUNCATE` rejected by triggers;
  three new indexes present.

### Commit 3 — T3 · `CursorCodec` + exceptions + handler bindings

New:
- `service/Cursor.java` — record `(int v, Instant ts, String id, String fh)`.
- `service/CursorCodec.java` — pure (no Spring); `Base64.getUrlDecoder/
  Encoder` + a private `ObjectMapper`.
- `service/CursorDecodeException.java` — `extends IllegalArgumentException`.
- `service/QueryValidationException.java` — `extends RuntimeException`;
  carries field-level messages compatible with the existing envelope's
  `errors: [...]` array.
- `src/test/java/com/sam/auditlog/service/CursorCodecTest.java`.

Modified:
- `controller/GlobalExceptionHandler.java` — two new
  `@ExceptionHandler`s: `CursorDecodeException` → 400, `QueryValidationException`
  → 422. Both reuse the existing envelope shape (`{timestamp, status, error,
  errors[]}`) — do not invent a new format.

### Commit 4 — T4 · Repository keyset query

Modified:
- `repository/AuditEventRepository.java` — add `findPage(...)` per design
  §6.3 (JPQL with null-tolerant filter predicates and the `(timestamp, id) <
  (cursorTs, cursorId)` keyset clause; `Pageable` carries `limit + 1`).

New:
- `src/test/java/com/sam/auditlog/repository/AuditEventRepositoryQueryIT.java`
  — Testcontainers integration. Each filter combo from §4 plus same-
  millisecond tiebreaker plus keyset slicing.

### Commit 5 — T5 · `AuditEventQueryService` + semantic validation

New:
- `service/QuerySpec.java` — internal record carrying `actor`, `resource`,
  `from`, `to`, decoded `Cursor`, `limit`. Lives under `service/` (not
  `dto/`) so it is not exposed at the API boundary.
- `service/AuditEventQueryService.java` — `@Service`. Validates (`from >=
  to`, `limit` range, empty filter strings, cursor `v`, cursor `fh` vs
  `CursorCodec.filterHash(currentFilters)`); calls `findPage` with
  `Pageable.ofSize(limit + 1)`; emits `next_cursor` from the *limit-th* row
  only when `limit + 1` rows were returned.
- `dto/AuditEventPage.java` — record `(List<AuditEventResponse> events,
  String nextCursor)`. `@JsonProperty("next_cursor")` + `@JsonInclude(NON_NULL)`
  so the field is omitted on the final page.
- `src/test/java/com/sam/auditlog/service/AuditEventQueryServiceTest.java`
  — pure unit. Hand-rolled stub repository (no Mockito; keeps the repo's
  minimal-test-deps pattern). Real `CursorCodec`. Covers every validation
  branch, has-more boundary at exactly `limit`, has-more at `limit + 1`,
  empty result, default-limit case.

### Commit 6 — T6 · `GET /api/v1/audit-events` + end-to-end coverage

Modified:
- `controller/AuditEventController.java` — add `@GetMapping query(...)`.
  `@RequestParam` for `actor`, `resource`, `cursor` (Strings); `from`, `to`
  bound as `Instant` (Spring binding handles RFC 3339 with offsets, normalizing
  to UTC); `limit` bound as `Integer` with `defaultValue = "50"`. Decodes
  cursor via `CursorCodec` (may throw `CursorDecodeException` → 400), builds
  `QuerySpec`, delegates to `AuditEventQueryService`.

New:
- `src/test/java/com/sam/auditlog/integration/AuditEventQueryIntegrationTest.java`
  — Testcontainers, organized by AC family per tasks.md T6: filters,
  response shape (microsecond precision passes through unchanged — no
  Jackson `Instant` formatter is configured, that is the contract), parse
  errors, semantic errors (including both asymmetric AC3.5 cases),
  ordering with ties, pagination stability under concurrent inserts, limit
  edges, opaque-cursor behavior at the HTTP boundary.

### Commit 7 — T7 · OpenAPI / springdoc annotations

Modified:
- `controller/AuditEventController.java` — `@Operation`, `@Parameter`,
  `@ApiResponse` on both handlers.
- DTOs under `dto/` — `@Schema` descriptions on non-obvious fields.

No behavior change; existing tests still pass.

## Verification

After every commit:
- `mvn verify` — unit, Testcontainers integration, and ArchUnit
  (`LayerBoundaryTest`) all green.
- `LayerBoundaryTest` requires no rule edits at any point — all new classes
  land in existing packages (`controller`, `service`, `repository`, `dto`,
  `converter`, `model`).

After Commit 6 (read-path milestone), exercise manually before opening the PR:
- Boot the app locally.
- `curl` the golden-path queries from `requirements.md`: no-filter page;
  `actor=…`; full walk via `next_cursor` until exhaustion (assert
  union-equals-seeded, no duplicates); `from >= to` → 422; tampered cursor
  → 400 (decode) or 422 (filter-hash).
- Capture one curl example in the PR description as a smoke trace.

After Commit 7:
- Open `/swagger-ui.html`; both endpoints render their full shapes.
- `GET /v3/api-docs` validates as a well-formed OpenAPI document.
